## 它跟 Spotlight 到底差在哪

很多人第一次听说 LaunchBar，反应都是"Spotlight 不够用吗？"

够用。但 LaunchBar 不是"更好的 Spotlight"，它是另一种东西。

Spotlight 的逻辑是搜索：你告诉它你要找什么，它帮你找。LaunchBar 的逻辑是**操作流**：你想做什么，它帮你做——而且可以一步接着一步，不需要切换任何窗口。

举个例子。你在 Finder 里看到一个文件，想发给某人。Spotlight 能帮你开 Finder，但接下来你还是得自己操作。而 LaunchBar 支持"即时传送（Instant Send）"——双击 Fn，文件直接飞进 LaunchBar，接着你选择动作，比如发邮件、移动到某个文件夹、压缩打包，回车，搞定。

这就是差距。Spotlight 是查询工具，LaunchBar 是操作工具。

## 几个让人上瘾的功能

**快速启动（当然）**

按下自定义快捷键唤出 LaunchBar，输入 app 名字的几个字母，回车，开。它有自适应学习算法，打多了就能记住你的习惯，比如你总是用"WE"打开 WeChat，它下次就会优先把微信排在最前面。

**Action 动作系统**

选中一个文件后，按 Tab 键，LaunchBar 就会问你"你想对它做什么"——发送给某人、压缩、用特定 app 打开……这套交互逻辑是整个软件最迷人的地方，一旦用顺了，你会觉得以前怎么忍的。

**剪贴板历史**

最多可以存 100 条剪贴板记录，还支持纯文本模式（复制啥都自动去格式），不需要再单独装一个剪贴板管理软件。

**Snippet 文本片段**

频繁要输的邮箱、签名、模板，存成 Snippet，以后输个几个字母就自动扩展出来。配合系统自带文本替换用，基本可以替代专门的文本扩展工具。

**自定义动作（进阶玩家专属）**

内置动作编辑器，支持 Shell、Python、Ruby、PHP 脚本，理论上你能实现任意自动化操作。这是专门给那种"恨不得把所有重复操作都用脚本干掉"的人准备的。

## 使用门槛说实话

LaunchBar 有学习成本，这是真的，不需要为它洗白。

它的交互逻辑跟普通软件不一样，需要花时间熟悉。比如它的输入框没有退格键——输错了不删，而是等一秒让系统重置，然后从头开始输。初看觉得有点奇怪，但用一段时间就会发现这样反而更快。

官方提供完整的英文文档，也有不少中文社区教程可以参考。通常一到两周能基本上手，一个月后会开始真的享受这套操作逻辑。

如果你平时用电脑比较多，对效率比较在意，这个学习成本是值的。如果你只是偶尔用电脑、对快捷操作没什么需求，那 Spotlight 已经够了。
